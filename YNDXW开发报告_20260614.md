### 核心代码逻辑

#### 1. 场景初始化 (`app.js`)
```javascript
(function() {
  'use strict';
  
  // SimplexNoise实现（修复浏览器解构赋值兼容性）
  class SimplexNoise {
    constructor() {
      this.p = new Uint8Array(256);
      for (let i = 0; i < 256; i++) this.p[i] = i;
      for (let i = 255; i > 0; i--) {
        const j = Math.floor(Math.random() * (i + 1));
        const t = this.p[i];  // 传统变量交换（避免浏览器SyntaxError）
        this.p[i] = this.p[j];
        this.p[j] = t;
      }
    }
  };
  
  // 地形生成（多层噪声）
  function buildTerrain() {
    const geometry = new THREE.PlaneGeometry(100, 100, segments, segments);
    const positions = geometry.attributes.position.array;
    for (let i = 0; i < positions.length; i += 3) {
      const x = positions[i];
      const z = positions[i + 1];
      positions[i + 2] = 
        noise.noise2D(x * 0.005, z * 0.005) * 8 +  // 大山脉
        noise.noise2D(x * 0.01, z * 0.01) * 3 +   // 中起伏
        noise.noise2D(x * 0.02, z * 0.02) * 1;    // 小细节
    }
    geometry.computeVertexNormals();
    return geometry;
  }
  
  // 初始化场景
  init();
})();
```

### 关键设计决策

#### 1. 为什么不用OrbitControls？
- **原因**：OrbitControls需要用户拖拽旋转，不符合"第一人称视角"需求
- **方案**：自定义鼠标控制（mousedown记录拖拽，计算yaw/pitch）
- **优势**：更简洁（无需额外JS文件），更符合产品定位

#### 2. 为什么用IIFE模式？
- **原因**：避免全局变量污染，所有变量封装在匿名函数内
- **方案**：`(function() { 'use strict'; ... })();`
- **优势**：代码更模块化，避免与第三方库冲突

#### 3. 为什么用Canvas纹理做树木？
- **原因**：3D树木模型（多层树冠）性能消耗大
- **方案**：低性能时用Canvas绘制树纹理 + Sprite替代
- **优势**：性能提升3-5倍，视觉差异不大

### 已知问题

#### 1. GitHub Pages国内访问不稳定
- **现象**：国内用户访问 https://13888285815.github.io/yndxw-homepage/ 偶尔超时
- **原因**：GitHub Pages服务器在国外，国内网络波动
- **方案**：部署到Gitee Pages（国内CDN）
- **状态**：Gitee remote已配置，需用户手动生成令牌推送

#### 2. 移动端小地图显示异常
- **现象**：小地图canvas 40x40px在移动端被放大模糊
- **原因**：CSS `image-rendering: pixelated` 在某些浏览器不生效
- **方案**：用SVG重绘小地图（矢量图形）
- **优先级**：低（不影响核心功能）

#### 3. 建筑点击热区不够精确
- **现象**：点击建筑边缘可能无法触发
- **原因**：Raycasting检测的是3D模型，热区是AABB包围盒
- **方案**：用多个小型BoxGeometry组合热区
- **优先级**：中（影响用户体验）
